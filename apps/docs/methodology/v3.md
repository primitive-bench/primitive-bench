## Principles

1. **No single winner.** Results are reported per slice and per constraint. The
   winner changes by slice — that is the product, not a caveat. (D-01)
2. **Real evals, not reviews.** Every published claim is backed by a citable
   statistical test with a confidence interval. (D-04)
3. **Reproducible by anyone.** Public dev splits + run manifests reproduce public
   runs bit-for-bit (deterministic seed, pinned versions). (D-14)
4. **Held-out answers never leak.** Test splits live behind the eval server with
   HMAC-keyed integrity; only scores come back. (D-07)
5. **Neutral arbiter.** No pay-to-rank. The disputed-LoCoMo-style market gaps are
   exactly where a neutral, reproducible benchmark wins trust. (D-17)

## Ground-truth tiers (D-09)

| Tier | Meaning | Use |
|------|---------|-----|
| `verified_external` | Human-verified against an external source | Highest-trust golden items |
| `authoritative_registry` | Canonical registry of record | Stable factual answers |
| `sentinel_planted` | Known item planted into the set | Drift + contamination detection |

## Statistical methods (D-04)

- **Paired comparison:** McNemar's test with continuity correction on discordant
  pairs `n01, n10`. χ² = (|n01−n10|−1)²/(n01+n10).
- **Proportion CIs:** Wilson score interval (preferred over CLT at eval-sized n).
- **Continuous metrics (nDCG, MCC, AUROC, latency):** seeded percentile bootstrap
  (1,000–10,000 resamples), seed recorded in the manifest.
- **Retrieval:** nDCG@k, MAP@k, MRR@k, Recall@k, hit@k (BEIR/MTEB standard).
- **Chunking:** downstream token-range recall / precision / IoU / precision_Ω against
  gold reference spans after chunk→embed→retrieve, with a fixed embedder held constant
  across chunkers (Chroma "Evaluating Chunking Strategies" methodology). The
  separability surface is the paired binary **coverage@k** (recall@k ≥ 0.8).
- **Ranking:** Bradley-Terry / Elo with bootstrap CIs (FastChat pattern).

## Separability gate (D-10)

A slice publishes a winner **only if** the top adapter is statistically separable
from the runner-up (McNemar p < α at the slice's n, CIs non-overlapping). If not:
increase n (derived from measured discordance) or merge the slice. Low separability
is a guardrail outcome, not a failure.

## Saturation guard (D-11)

If a golden set exceeds ~90% with minimal spread, it is exhausted — rotate in fresh
sentinel-planted items. (SimpleQA saturation lesson.)
